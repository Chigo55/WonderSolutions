# Development Workflow Meta-Rule

> Related meta-rules: `${CLAUDE_PLUGIN_ROOT}/rules/backend.md` · `${CLAUDE_PLUGIN_ROOT}/rules/frontend.md` · `${CLAUDE_PLUGIN_ROOT}/rules/security.md`

This document defines the **wonder-harness 6-stage workflow**. It applies to every development task regardless of stack.

## The 6-Stage Pipeline (fixed order, no skipping)

| Stage | Agent | Input | Output |
|-------|-------|-------|--------|
| 1. Analysis | analyzer | task definition + `.claude/rules/` | `work-doc.md §Analysis` |
| 2. Research | researcher | §Analysis | `work-doc.md §Research` |
| 3. Planning | planner | §Analysis + §Research | `work-doc.md §Planning` |
| 4. Implementation | developer | §Planning + `.claude/rules/` | code files |
| 5. Inspection | inspector | code + §Planning + `.claude/rules/` | `inspection-report.md` |
| 6. Modification | modifier | `inspection-report.md` | `modification-report.md` + delegated fixes |

## Enforcement

- In `/wh-run` command mode: hooks enforce stage ordering. Writing the wrong artifact type for the current stage is blocked.
- In prompt mode: this workflow is recommended but not enforced. The assistant should follow it proactively.

## Project Rules as the Stack Source

Each agent reads `.claude/rules/` to determine stack-specific behavior:
- `.claude/rules/backend.md` — server-side conventions
- `.claude/rules/frontend.md` — client-side conventions
- `.claude/rules/security.md` — security policies

These files are generated by `/wh-init` from the project's actual codebase. They are the authoritative source — not the plugin's meta-rules.

## Deliverable Structure

```
.claude/
└── runs/{run-id}/
    ├── work-doc.md           ← §Analysis, §Research, §Planning
    ├── inspection-report.md  ← quality, correctness, security, rules findings
    └── modification-report.md  ← fix list and applied changes (Stage 6 only)
```

## Principles

- Every stage must produce its deliverable before the next stage begins.
- YAGNI: implement only what the task specifies.
- Project rules override general knowledge. If `.claude/rules/backend.md` says X, do X.
