enforce-template

세션 내 템플릿 탐색이 없으면 코드 생성을 차단한다. templater 단계 선행을 강제하는 주요 집행 메커니즘.

개요

모든 Write·Edit 호출 전 세션 마커를 확인한다. 마커가 없으면 permissionDecision: "deny"를 반환해 쓰기를 차단한다. 차단 — 실제 쓰기 실행 안 됨.

현재 동작

  • 모든 Write·Edit 호출 전 트리거.
  • decide.js로 .claude/.state/state.json의 세션 마커 확인.
  • 마커 없음: deny + 메시지 반환.
  • 마커 있음: allow 반환.
  • 차단된 쓰기는 실행되지 않는다. 타임아웃 10초.

입력 / 출력

내용
입력PreToolUse 이벤트 (대상 파일 경로 포함)
출력{ permissionDecision: "allow" | "deny", denyMessage?: string }

의존성

  • mark-template-read.js 훅이 같은 세션 내에서 마커를 먼저 채워야 한다.
  • lib/decide.js — 결정 로직.
  • lib/state.js — 상태 파일 접근.
  • .claude/templates/index.json — 안내 메시지용 참조.

개선 방향

거부 메시지에 읽어야 할 특정 템플릿 이름 명시 (예: "eq-controller 템플릿을 먼저 읽으세요").
"아무 템플릿도 없음" vs "잘못된 템플릿 유형" 구분 — 더 정밀한 안내.
템플릿 없이 부트스트랩해야 하는 경우를 위한 --force 탈출구 제공.
설정 파일·.gitkeep 등 템플릿 불필요한 경로를 화이트리스트 처리.
