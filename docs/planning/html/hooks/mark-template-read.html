mark-template-read

Read 툴 호출 후 해당 파일이 템플릿인지 확인하고 세션 마커를 기록한다. enforce-template 훅의 판단 근거를 제공한다.

개요

모든 Read 이후 읽은 파일이 index.json의 pathPatterns와 일치하면 .claude/.state/state.json에 마커를 기록한다. 비차단 — 결과에 영향 없음.

현재 동작

  • 모든 Read 툴 호출 후 트리거.
  • 읽은 파일 경로를 index.json pathPatterns와 대조 (index-match.js).
  • 일치하면 marker.js를 통해 세션 마커 기록.
  • 비차단; 툴 호출 결과 또는 파라미터 수정 없음. 타임아웃 5초.

입력 / 출력

내용
입력PostToolUse 이벤트 (파일 경로 파라미터)
출력.claude/.state/state.json에 마커 항목 기록

의존성

  • .claude/templates/index.json — 패턴 매칭 기준.
  • lib/index-match.js — glob 패턴 매칭 로직.
  • lib/marker.js — 마커 읽기/쓰기.
  • lib/state.js — 상태 파일 관리.

개선 방향

파일 열기만으로 마킹되는 현재 방식 대신 최소 읽기 깊이(예: 10행 이상) 조건 추가.
"어떤 템플릿이라도 읽었음" 대신 특정 템플릿 ID 단위로 추적.
/wh-status 커맨드에서 탐색 완료·미완료 템플릿 목록 노출.
