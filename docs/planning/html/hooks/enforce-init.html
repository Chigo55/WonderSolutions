enforce-init

/wh-init 실행 중 ADR → 규칙 → 보고서 순서를 강제한다. 순서를 건너뛰면 쓰기를 차단한다.

개요

ruler 에이전트가 순서를 지키도록 단계별 쓰기 대상 경로를 검사한다.

① adr-extract
.claude/adr/
→
② generate
.claude/rules/
→
③ report
.claude/reports/

현재 동작

  • /wh-init 세션 중 모든 Write·Edit 호출 전 트리거.
  • init-guard.js로 .claude/.state/state.json의 진행 상태 확인.
  • .claude/adr/ 없이 .claude/rules/ 쓰기 시도: deny.
  • .claude/rules/ 없이 .claude/reports/ 쓰기 시도: deny.
  • 순서 충족 시: allow. 타임아웃 10초.

입력 / 출력

내용
입력PreToolUse 이벤트 (대상 파일 경로) + 현재 상태
출력{ permissionDecision: "allow" | "deny", denyMessage?: string }

의존성

  • lib/init-guard.js — 단계 순서 검증 로직.
  • lib/state.js — 초기화 진행 상태 읽기.
  • ruler 에이전트가 각 단계 완료 후 상태를 업데이트해야 한다.

개선 방향

--reset 플래그로 init 상태 초기화 후 1단계부터 깔끔하게 재실행.
ADR·규칙이 이미 있을 때 3단계(보고서)만 재실행하는 부분 재실행 지원.
거부 메시지에 시도된 단계와 기대 단계를 명시해 디버깅 용이하게.
모든 엣지 케이스(순서 이탈, 파일 없음, 부분 상태)에 대한 단위 테스트 보강.
