ruler

산출물을 규칙 체크리스트로 교차 검증하고, ADR에서 프로젝트별 규칙을 생성하며, HTML 보고서를 작성한다.

개요

파이프라인 종단(review 모드)과 초기화(/wh-init) 두 역할을 겸한다. 5가지 동작 모드를 지원한다.

review adr-extract generate create modify

현재 동작

모드동작
review구현된 7파일을 레이어별 체크리스트로 검증. CRITICAL 항목 시 완료 차단.
adr-extract기존 코드베이스 스캔 → .claude/adr/{layer}.md 생성 (Context/Decision/Rationale/Consequences).
generateADR + 메타 규칙 → .claude/rules/{layer}.md 생성.
create신규 메타 규칙 문서 작성.
modify기존 규칙·ADR 갱신.

입력 / 출력

모드입력출력
review파이프라인 전체 산출물검증 보고서 (체크리스트 항목별 통과/실패)
adr-extract기존 코드베이스.claude/adr/{layer}.md
generateADR + 메타 규칙.claude/rules/{layer}.md

의존성

  • enforce-init.js 훅: adr-extract → generate → report 순서 강제.
  • generate 모드 전에 ADR이 반드시 존재해야 한다.
  • 메타 규칙이 없으면 generate 모드 실행 불가 (폴백 없음).

개선 방향

CRITICAL/HIGH/MEDIUM/LOW 심각도 분류 + LOW 자동 수정.
--strict 플래그: MEDIUM 항목도 차단 처리.
증분 규칙 갱신: 변경된 ADR 섹션만 재생성.
HTML 보고서에 규칙-코드 추적 가능성 매트릭스 포함.
검증 보고서를 JSON으로 내보내 CI 파이프라인 연동 지원.
