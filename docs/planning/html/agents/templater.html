templater

템플릿 카탈로그(index.json)를 관리하고 developer가 코드 작성 전 적절한 템플릿을 탐색했는지 보장하는 단계.

개요

모듈 계획의 각 파일에 대해 카탈로그에서 매칭 템플릿을 탐색한다. 없으면 기존 코드베이스에서 패턴을 추출해 신규 등록한다.

토큰 컨벤션 (authoritative):  {module} {domainName} Xxx xxx {gridId} {Entity}

현재 동작

index.json 로드
→
pathPatterns 매칭
→
매칭 있음: 읽기 지시
매칭 없음
→
기존 코드에서 추출
→
index.json 신규 등록
  • 모든 템플릿은 구문 오류 없이 검증된 후 developer에게 전달된다.
  • modify 모드: 변경 파일 관련 템플릿만 확인.
  • review 모드: 카탈로그 누락·오래된 패턴 감사.

입력 / 출력

내용
입력planner의 모듈 계획 + .claude/templates/index.json
출력갱신된 index.json + developer를 위한 템플릿 탐색 지시

의존성

  • .claude/templates/index.json이 존재해야 한다 (init-requests 훅으로 시드).
  • mark-template-read.js 훅이 세션 내 읽은 템플릿을 추적한다.
  • enforce-template.js 훅이 이 단계 완료를 developer 쓰기의 전제 조건으로 강제한다.
  • templates.md 메타 규칙이 토큰 컨벤션의 권위 있는 출처.

개선 방향

기존 코드베이스에서 템플릿 후보를 수동 추출 없이 자동 탐지.
기존 템플릿과 현재 구현의 diff 뷰 제공 — 갱신 여부 결정 지원.
index.json에 템플릿 버전 관리 추가 — 구버전 보존 가능.
developer 전달 전 토큰 치환 완전성 검증 (미치환 {token} 탐지).
