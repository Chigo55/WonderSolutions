developer

계획과 템플릿을 바탕으로 실제 Java·Thymeleaf·ES6 코드를 작성한다. 프로젝트 규칙과 메타 규칙의 이중 제약 아래 동작한다.

개요

플래너가 결정한 순서대로 7파일 세트를 구현한다. enforce-template 훅이 활성화되어 있어 템플릿 탐색 없이 코드 작성이 차단된다.

현재 동작

백엔드 강제 규칙

  • Controller → Service → Mapper 단방향 의존성만 허용.
  • SP 전용 데이터 접근 (@Insert/@Update/@Delete 금지).
  • 모든 CUD 서비스 메서드에 @Transactional 필수.
  • 패키지 루트: io.boot.wonder.web.

프론트엔드 강제 규칙

  • Thymeleaf + Kendo 웹 컴포넌트 (JSP·jQuery 금지).
  • ES6 모듈; 전역 상태 금지.
  • 모든 데이터 변경에 권한 가드 필수.

입력 / 출력

내용
입력모듈 계획 + 템플릿 탐색 지시 + 프로젝트 규칙
출력프로젝트 소스 트리에 작성된 Java/HTML/JS 파일

의존성

  • enforce-template.js 훅: templater 단계 생략 시 쓰기 차단.
  • 프로젝트 규칙(.claude/rules/)이 메타 규칙보다 우선; 둘 다 준수.
  • SP 정의가 DB에 미리 존재해야 한다 (developer가 SP를 생성하지 않음).

개선 방향

구현된 각 파일과 함께 단위 테스트 스텁 자동 생성.
mapper 호출 작성 전 참조 SP가 DB 스키마에 존재하는지 검증.
modify 모드에서 파일 전체 재작성 대신 최소 diff 적용.
쓰기 후 Bash 툴로 즉시 컴파일 오류 확인.
