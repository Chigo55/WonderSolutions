Hooks

사용자 개입 없이 워크플로우 규율을 자동으로 강제하는 이벤트 기반 훅 4개. Node.js 빌트인만 사용하는 경량 스크립트.

개요

훅이벤트스크립트차단 여부
init-requestsSessionStartinit-requests.js비차단
mark-template-readPostToolUse (Read)mark-template-read.js비차단
enforce-templatePreToolUse (Write|Edit)enforce-template.js차단
enforce-initPreToolUse (Write|Edit)enforce-init.js차단

현재 동작

  • init-requests: 세션 시작 시 요청 양식 시드 파일을 자동 복사한다.
  • mark-template-read: 세션 중 읽은 템플릿 파일을 수동적으로 추적한다.
  • enforce-template: 템플릿 탐색 없이 Write/Edit 시 접근을 거부한다.
  • enforce-init: /wh-init 중 ADR → 규칙 → 보고서 순서를 강제한다.
  • 공유 라이브러리(hooks/scripts/lib/)가 상태·마커·결정 로직을 담당한다.
  • 상태는 .claude/.state/state.json에 세션 내 지속 저장된다.

입력 / 출력

훅입력출력 / 효과
init-requests세션 시작 이벤트시드 파일 → .claude/requests/ 복사
mark-template-readRead 툴 호출 파라미터.claude/.state/에 마커 기록
enforce-templateWrite/Edit 툴 호출 파라미터allow 또는 deny 결정
enforce-initWrite/Edit 툴 호출 파라미터allow 또는 deny 결정

의존성

  • lib/state.js — 모든 훅이 공유하는 상태 파일 관리.
  • lib/marker.js — 템플릿 읽기 추적.
  • lib/decide.js — 템플릿 강제 결정 로직.
  • lib/init-guard.js — 단계 순서 검증 로직.
  • .claude/templates/index.json — 템플릿 패턴 매칭 기준.

개선 방향

enforce-template 거부 메시지에 읽어야 할 구체적인 템플릿 이름 안내.
mark-template-read에 최소 읽기 깊이(예: 10행 이상) 조건 추가.
developer 코드 작성 후 PostToolUse에서 린트/포맷 자동 실행 훅 추가.
init-requests에 기존 요청 파일 만료 감지(N일 경과) 후 갱신 제안 기능.
세션 재시작 없이 .claude/.state/state.json을 초기화하는 --reset 플래그 지원.
