Command · 최초 1회 초기화

/wh-init

기존 코드베이스에서 아키텍처 결정을 역공학으로 추출하고 프로젝트별 규칙과 HTML 보고서를 레이어별로 생성하는 초기화 커맨드.

개요

/wh-create·/wh-modify 실행 전에 먼저 실행해야 최적의 결과를 얻을 수 있다. 레이어 플래그로 초기화 범위를 지정한다.

--backend --frontend --security --templates

현재 동작

각 레이어에 대해 아래 순서로 순차 실행 (enforce-init 훅으로 순서 강제):

① adr-extract
ADR 추출
→
② generate
규칙 생성
→
③ report
HTML 보고서

입력 / 출력

내용
입력레이어 플래그 + 기존 코드베이스
출력 .claude/adr/{layer}.md
.claude/rules/{layer}.md
.claude/reports/wh-init-{layer}-YYYYMMDD-HHMMSS.html

의존성

  • ADR 추출을 위해 기존 Java/JS/HTML 파일이 읽기 가능해야 한다.
  • 메타 규칙 파일(plugins/wonder-harness/rules/{layer}.md)이 존재해야 한다.
  • enforce-init.js 훅이 단계 순서를 강제하며 상태는 .claude/.state/state.json에 저장된다.

개선 방향

--all 플래그로 모든 레이어를 한 번에 초기화.
코드베이스 대규모 변경 감지 후 재초기화 권유 기능.
증분 모드: 마지막 초기화 이후 변경된 섹션만 ADR·규칙 갱신.
HTML 보고서에 메타 규칙 vs 프로젝트 규칙 나란히 비교 diff 포함.
--reset 플래그로 상태 초기화 후 1단계부터 재실행.
