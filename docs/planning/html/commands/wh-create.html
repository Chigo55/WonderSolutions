/wh-create

신규 도메인 기능을 처음부터 생성하는 진입 커맨드. 요청 문서를 검증하고 전체 파이프라인을 실행한다.

개요

.claude/requests/create_request.md를 검증한 후 planner → templater → developer → ruler 파이프라인을 create 모드로 실행한다. 결과로 도메인 7파일 세트와 ruler 검증 보고서가 생성된다.

현재 동작

1
create_request.md 읽기.
2
필수 섹션 검증 — 누락 시 즉시 실패.
3
planner에 mode=create 전달.
4
planner → templater → developer → ruler 순서로 실행.
5
단계별 요약 + ruler 최종 검증 보고서 출력.

입력 / 출력

내용
입력.claude/requests/create_request.md (필수 4섹션 포함)
출력7파일 도메인 모듈 (Java 5 + 프론트엔드 2) + ruler 검증 보고서

필수 요청 섹션:

## Goal ## Scope ## Constraints ## Acceptance Criteria

의존성

  • init-requests.js 훅이 .claude/requests/에 시드 파일을 복사해야 한다.
  • 4개 에이전트 모두 플러그인에 존재해야 한다.
  • developer 단계 중 enforce-template.js 훅이 활성화된다.

개선 방향

--dry-run 플래그: planner 계획만 출력하고 코드 생성 없이 종료.
--layer 플래그: 백엔드 또는 프론트엔드만 생성하는 부분 실행 지원.
완료 요약에 생성된 모든 파일 경로와 라인 수 포함.
