Command · review 모드

/wh-review

create/modify 파이프라인 없이 기존 코드·템플릿·규칙을 단독으로 리뷰하는 커맨드.

개요

리뷰 대상에 따라 적합한 에이전트를 선택해 실행한다. 파일은 작성하지 않으며 사용자가 명시적으로 수정을 요청할 때만 적용한다.

현재 동작

1
대상(파일 경로·도메인명·레이어)을 인수로 받는다.
2
리뷰 유형 결정 및 에이전트 선택:
코드 → developer   템플릿 → templater   규칙 → ruler
3
에이전트가 대상을 읽고 관련 체크리스트를 적용해 발견 사항 반환.
4
사용자 요청 없이 파일 수정 없음.

입력 / 출력

내용
입력대상 지정자 (파일 경로, 도메인명, 레이어 플래그)
출력리뷰 발견 사항 보고서 (문제점·심각도·개선 권장)

의존성

  • 대상 파일이 읽기 가능해야 한다.
  • 컨텍스트용 관련 규칙 파일(.claude/rules/{layer}.md)이 로드된다.
  • 요청 파일 불필요.

개선 방향

developer + ruler + security 에이전트 병렬 실행으로 다각도 리뷰 제공.
CRITICAL / HIGH / MEDIUM / LOW 심각도 분류 및 파일:라인 참조 포함 구조화 보고서.
--fix 플래그: 사용자 확인 후 LOW/MEDIUM 발견 사항 자동 수정.
전체 도메인(7파일) 일괄 리뷰 지원.
git diff 연동 — 마지막 커밋 이후 변경된 파일만 리뷰.
