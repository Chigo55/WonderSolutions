/wh-modify

기존 도메인 기능을 변경하는 진입 커맨드. 변경 대상을 명확히 지정하고 파이프라인을 modify 모드로 실행한다.

개요

modify_request.md를 검증하고 planner → templater → developer → ruler 파이프라인을 modify 모드로 실행한다. 모든 에이전트는 변경 대상 파일로 범위를 제한한다.

현재 동작

1
modify_request.md 읽기.
2
필수 섹션 검증 — 누락 시 즉시 실패.
3
planner가 변경 대상 파일 식별 (7파일 중 일부).
4
templater가 변경 관련 템플릿 확인·등록.
5
developer가 대상 파일만 수정.
6
ruler가 수정된 파일 범위 검증.

입력 / 출력

내용
입력.claude/requests/modify_request.md (필수 4섹션 포함)
출력수정된 파일 + ruler 검증 보고서 (변경 범위 한정)

필수 요청 섹션:

## Target ## Changes ## Impact ## Acceptance Criteria

의존성

  • 변경 대상 도메인이 이미 존재해야 한다 (planner가 읽을 수 있어야 함).
  • 수정 시에도 enforce-template.js 훅이 적용된다.
  • 기존 .claude/rules/{layer}.md가 개발자의 변경 범위를 제한한다.

개선 방향

developer 작성 전 계획된 변경 사항의 diff 미리보기 제공.
## Impact 섹션을 호출자 grep 스캔으로 자동 채우는 기능 추가.
단일 요청에서 다중 도메인 변경 지원 (현재 단일 도메인 가정).
ruler 검증 후 수정된 파일을 에디터에서 자동으로 열기.
