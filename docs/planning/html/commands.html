Commands

사용자가 직접 호출하는 4개의 슬래시 커맨드. 파이프라인의 진입점이며 각각 요청 파일 검증 후 에이전트를 위임한다.

개요

요청 문서를 검증하고 에이전트 파이프라인에 위임하는 마크다운 기반 진입점 커맨드 4개.

커맨드모드목적
/wh-createcreate신규 도메인 기능 생성
/wh-modifymodify기존 기능 변경
/wh-reviewreview단독 코드·템플릿·규칙 리뷰
/wh-initinit프로젝트 최초 초기화

현재 동작

  • 모든 커맨드는 필수 섹션 누락 시 즉시 실패(fail-fast) 처리한다.
  • /wh-create·/wh-modify는 planner → templater → developer → ruler 전체 파이프라인을 실행한다.
  • /wh-review는 대상에 맞는 에이전트를 단독 실행한다.
  • /wh-init는 레이어별로 adr-extract → generate → report 순서를 강제한다.
사용자 입력 → 커맨드 검증 → planner → templater → developer → ruler

입력 / 출력

커맨드입력출력
/wh-createcreate_request.md7파일 도메인 모듈 + ruler 검증 보고서
/wh-modifymodify_request.md수정된 파일 + ruler 검증 보고서
/wh-review대상 파일 또는 도메인명리뷰 발견 사항 보고서
/wh-init레이어 플래그ADR + 규칙 파일 + HTML 보고서

의존성

  • init-requests.js 훅이 세션 시작 시 시드 파일을 복사해야 한다.
  • 4개 에이전트(planner, templater, developer, ruler) 모두 플러그인에 존재해야 한다.
  • /wh-init은 ADR 추출을 위해 기존 코드베이스가 필요하다.

개선 방향

/wh-create·/wh-modify에 --dry-run 플래그 추가 — 플래너 계획만 출력하고 코드 생성 생략.
/wh-review에 병렬 멀티 에이전트 리뷰 지원 (developer + ruler + security 동시 실행).
/wh-init에 코드베이스 변경 감지 후 재초기화 권유 기능 추가.
파이프라인 단계 간 진행 표시자 제공 ("2/4단계: templater 실행 중...").
