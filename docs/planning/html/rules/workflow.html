workflow.md

도메인 네이밍 공식·규칙 적용 우선순위·템플릿 탐색 프로세스를 정의한다. 생성 산출물 없이 직접 참조되는 규칙.

개요

developer 에이전트가 직접 읽는 프로세스 규칙. ruler generate로 프로젝트별 파일이 생성되지 않는다 (참조 전용).

현재 동작

도메인 네이밍 공식
모듈코드(2자 소문자) + camelCase 기능명 = 도메인명
예: wo + WorkShift = woWorkShift → 클래스 WoWorkShift → URL /wo/workShift
  • 규칙 우선순위: .claude/rules/ > 플러그인 메타 규칙.
  • developer는 .claude/rules/ 먼저 확인.
  • templater 완료 전 developer 코드 작성 금지.
  • DB 레이블 등록 컨벤션 (SP 명명·카탈로그 등록).

입력 / 출력

내용
입력developer·ruler 에이전트가 직접 읽음
출력— (참조 전용, 생성 산출물 없음)

의존성

  • developer 에이전트의 주 참조 소스 (네이밍·프로세스).
  • ruler가 네이밍 일관성 검증 시 참조.
  • templates.md가 토큰 컨벤션을 보완.

개선 방향

2자 모듈 코드 충돌 시 대응 가이드 추가.
자주 발생하는 네이밍 실수 트러블슈팅 섹션 추가.
표준 워크플로우 체크리스트에 DB 레이블 등록 단계 명시.
규칙 적용 우선순위를 순서도로 시각화.
프로젝트별 프로세스 편차를 기록하는 프로젝트 전용 workflow.md 생성 고려.
