Wonder Harness/Rules/backend
메타 규칙 · Java / Spring Boot / MyBatis

backend.md

Java·Spring Boot·MyBatis 레이어의 아키텍처 제약과 컨벤션을 정의한다. ruler가 이를 기반으로 프로젝트별 규칙을 생성한다.

개요

ruler generate 모드가 ADR + 이 메타 규칙을 조합해 .claude/rules/backend.md를 생성한다. 프로젝트별 규칙이 있으면 메타 규칙보다 우선 적용된다.

현재 동작

포함 섹션:

  • 레이어 구조 Controller → Service → Mapper 단방향 의존성
  • 네이밍 PascalCase 클래스, camelCase 메서드, 2자 모듈 접두사
  • 데이터 접근 SP 전용 (@Select + CALL sp_*), @Insert/@Update/@Delete 금지
  • DTO/Form 입력(Form)·출력(DTO) 분리; 엔티티 직접 노출 금지
  • 서비스 조회 @Transactional(readOnly=true)
  • 서비스 CUD @Transactional 필수
  • 컨트롤러 API는 @RestController, 뷰는 @Controller; @PreAuthorize 전 엔드포인트

입력 / 출력

내용
입력.claude/adr/backend.md + 이 메타 규칙
출력.claude/rules/backend.md (프로젝트별 규칙)

의존성

  • ruler generate 모드가 실행 에이전트.
  • .claude/adr/backend.md가 먼저 있어야 한다.
  • 프로젝트별 규칙이 이 메타 규칙보다 우선.

개선 방향

SP 호출 패턴 및 파라미터 바인딩 예시 보강.
전역 @ControllerAdvice vs 메서드별 try/catch 에러 처리 패턴 추가.
페이지네이션 컨벤션 (page/size 파라미터, PageRequest 사용법) 추가.
API 엔드포인트의 표준 응답 envelope 포맷 정의.
레이어별 테스트 컨벤션 (단위 vs 통합) 추가.
