templates.md

템플릿 파일의 토큰 컨벤션과 index.json 카탈로그 스키마를 정의하는 권위 있는 참조 규칙.

개요

templater와 developer가 직접 읽는다. 생성 산출물 없음 (참조 전용). 토큰 컨벤션의 권위 있는 출처.

{module} {domainName} Xxx xxx {gridId} {Entity}

현재 동작

토큰의미예시
{module}2자 소문자 모듈 코드wo, eq, inv
{domainName}camelCase 도메인명woWorkShift
XxxPascalCase 클래스 플레이스홀더WoWorkShift
xxxcamelCase 변수 플레이스홀더woWorkShift
{gridId}HTML 그리드 요소 IDwoWorkShiftGrid
{Entity}엔티티 PascalCase명WorkShift
  • 모든 템플릿에 토큰과 기대값을 나열하는 치환 테이블 주석 포함 필수.
  • 섹션 구분자 주석 컨벤션 표준화 (헤더·본문·풋터).
  • index.json 항목 구조: id, pathPatterns, description, path, metadata.

입력 / 출력

내용
입력templater·developer 에이전트가 직접 읽음
출력— (참조 전용)

의존성

  • templater가 토큰 컨벤션의 주요 소비자.
  • developer가 치환 시 이 컨벤션을 따른다.
  • index.schema.json (JSON Schema draft-07)이 카탈로그 구조를 런타임 강제.

개선 방향

템플릿 버전 관리: index.json 항목에 "version" 필드 추가 — 오래된 템플릿 플래그 가능.
쓰기 전 미치환 {token} 플레이스홀더 스캔 표준 단계 정의.
레이어별 토큰 적용 범위 표 추가 (어떤 파일 유형에 어떤 토큰).
선택적 섹션 처리 가이드 (조건부 토큰 치환) 추가.
플러그인 버전 간 토큰명 변경 시 마이그레이션 가이드 추가.
