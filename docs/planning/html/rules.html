Rules

대상 스택의 아키텍처 제약과 컨벤션을 정의하는 5개 메타 규칙 문서. ruler 에이전트가 이를 기반으로 프로젝트별 규칙을 생성한다.

개요

규칙레이어생성 산출물
backend.mdJava / Spring Boot / MyBatis생성됨 .claude/rules/backend.md
frontend.mdThymeleaf / Kendo / ES6생성됨 .claude/rules/frontend.md
security.md전 레이어 공통생성됨 .claude/rules/security.md
workflow.md프로세스 / 네이밍참조 전용
templates.md토큰 컨벤션 / 카탈로그참조 전용

현재 동작

  • 메타 규칙은 각 에이전트가 검증할 기준 체크리스트를 제공한다.
  • 프로젝트별 규칙(.claude/rules/)이 존재하면 메타 규칙보다 우선 적용된다.
  • backend.md: SP 전용 데이터 접근 강제 (@Insert/@Update/@Delete 금지).
  • frontend.md: Kendo 웹 컴포넌트 + ES6 모듈 강제 (JSP/jQuery 금지).
  • security.md: 입력 검증·권한 가드·파일 업로드·세션 처리 규칙.
규칙 우선순위
  1. .claude/rules/{layer}.md — 프로젝트별 규칙 (최우선)
  2. plugins/wonder-harness/rules/{layer}.md — 메타 규칙 (폴백)

입력 / 출력

규칙사용 에이전트출력
backend.mdruler (generate), developer (리뷰).claude/rules/backend.md
frontend.mdruler (generate), developer (리뷰).claude/rules/frontend.md
security.mdruler (generate), developer (리뷰).claude/rules/security.md
workflow.mddeveloper (참조)—
templates.mdtemplater, developer (참조)—

의존성

  • 규칙 생성 전에 ADR 문서(.claude/adr/)가 먼저 존재해야 한다.
  • ruler 에이전트가 규칙 생성 라이프사이클을 관리한다.
  • developer·ruler 에이전트 모두 파이프라인 실행 중 생성된 규칙을 읽는다.

개선 방향

security.md에 각 항목의 OWASP Top 10 매핑 추가.
backend.md에 SP 호출 패턴 및 트랜잭션 범위 예시 보강.
frontend.md에 Kendo 위젯별 초기화·destroy 패턴 추가.
workflow.md에 네이밍 실수 트러블슈팅 섹션 추가.
스택 테스트 컨벤션을 다루는 testing.md 메타 규칙 신규 추가.
규칙 버전 관리 지원 — 프로젝트가 특정 메타 규칙 버전에 고정 가능.
