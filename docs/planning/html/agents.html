Agents

순차 실행되는 4단계 파이프라인 에이전트. 각 단계의 출력이 다음 단계의 입력이 된다.

개요

1단계planner
→
2단계templater
→
3단계developer
→
4단계ruler

모든 에이전트는 create / modify / review 세 가지 모드를 지원한다.

현재 동작

  • 에이전트는 순차 실행된다 — 파이프라인 내 병렬 실행 없음.
  • 각 에이전트는 이전 단계 산출물(계획·템플릿·코드)을 컨텍스트로 읽는다.
  • ruler는 항상 최종 단계이며 검증 보고서를 생성한다.
  • 커맨드에서 전달된 모드에 따라 각 에이전트가 동작을 달리한다.

입력 / 출력

에이전트입력출력
planner요청 문서모듈 계획 (7파일 분해·의존성·순서)
templater모듈 계획갱신된 index.json, 템플릿 탐색 지시
developer계획 + 템플릿 + 규칙구현된 Java/HTML/JS 파일
ruler전체 산출물검증 보고서, ADR, 프로젝트 규칙

의존성

  • 모든 에이전트가 .claude/templates/index.json을 읽는다 (developer는 훅으로 강제).
  • 프로젝트 규칙(.claude/rules/)이 존재하면 메타 규칙보다 우선 적용된다.
  • developer는 enforce-template 훅에 의해 템플릿 탐색 없이 코드 작성이 차단된다.

개선 방향

developer 단계 내 독립 하위 작업의 병렬 실행 지원.
planner가 유사한 기존 모듈을 감지하고 재사용을 권장하는 기능 추가.
templater가 기존 코드베이스에서 템플릿 후보를 자동 탐지하는 기능 추가.
ruler의 발견 사항을 심각도별(CRITICAL/HIGH/MEDIUM)로 분류하고 LOW는 자동 수정.
단계 간 산출물 전달 검증 — 상위 단계 출력이 비정상일 경우 다음 단계에서 거부.
