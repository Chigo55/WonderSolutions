grill-me

계획·설계의 모든 분기가 해소될 때까지 한 번에 하나씩 집요하게 질문한다. 성급한 구현을 막는 기획 전 규율 강제 스킬.

개요

핵심 규칙: 질문 하나씩 → 권장 답변 제시 → 사용자 답변 수신 → 다음 분기로 이동.
코드베이스에서 객관적으로 답할 수 있는 질문은 사용자에게 묻기 전에 직접 탐색한다.

현재 동작

  • /grill-me <계획 설명>으로 호출.
  • Glob·Grep·Read로 객관적 질문 먼저 답변.
  • 한 번에 하나의 질문, 매번 권장 답변 제시.
  • 결정 트리 분기를 의존성 순서에 따라 해소.
  • 모든 미결 분기 해소 또는 사용자 종료 신호까지 계속.
  • 파일 작성 없음 — 대화 출력만.

입력 / 출력

내용
입력계획 또는 설계 설명 (인수)
출력Q&A 세션 → 설계에 대한 공유 이해

의존성

  • 대화 컨텍스트 의존 (계획 세부 사항).
  • 객관적 질문 탐색용으로 Glob·Grep·Read 사용.
  • 상태 파일 의존성 없음.

개선 방향

해소된 Q&A를 자동으로 구조화 기획 문서(docs/planning/{topic}.md)로 저장.
--scope 플래그로 특정 분기만 질문 (예: --scope security).
결정 트리에서 순환 의존성 감지 및 명시적 표시.
/wh-create와 연동 — 파이프라인 시작 전 grill-me 자동 실행.
